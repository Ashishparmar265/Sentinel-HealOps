high-speed telemetry and the "Control Plane" manages AI-driven decision-making.

3. CI/CD & DevSecOps Pipeline
The pipeline is designed for Canary Deployments with automated safety gates.
●​ Security: Gitleaks scans for secrets; Trivy scans Docker images for vulnerabilities.​
●​ Automation: GitHub Actions triggers a deployment to a 5% canary group.​
●​ The "Heal" Trigger: If the HealOps brain detects a $Z$-score anomaly in the canary, it
sends a webhook to GitHub to execute a helm rollback instantly.​

4. Authentication and Security
●​ Identity: Uses OIDC (OpenID Connect) for passwordless authentication between
GitHub Actions and Azure (Workload Identity Federation).​
●​ Secrets: All API keys for LLMs and databases are stored in Azure Key Vault and injected
at runtime.​
●​ Security Policy: Implements Zero-Trust networking where agents only communicate
over encrypted mTLS tunnels.

5. Frontend: The Sentinel Dashboard
The frontend is a React-based "Command Center" built with Tailwind CSS.
●​ Real-time Monitoring: WebSockets stream the AI's "Thought Process" and system health
metrics.
●​ Trace Viewer: A visual map showing every tool call (MCP) and agent-to-agent message
(ACP).
●​ Chaos Panel: A testing suite to manually inject faults (e.g., killing a tool server) to verify
self-healing.

6. Integration: Merging with SentinelARC
To merge HealOps with your existing SentinelARC project, follow this three-step integration
plan:

1.​ Instrumentation (The Sidecar):​
Modify your SentinelARC deployment to include the HealOps C++ Ingestor as a Sidecar
container in every agent pod. This allows the ingestor to read the agent's logs directly
from /dev/stdout.
2.​ Protocol Alignment:​
Update the Agent Communication Protocol (ACP) to include a "Metadata Header" in
every message. This header contains a trace_id that HealOps uses to track a request as it
moves from the Orchestrator to the Search Agent.
3.​ The Remediation Loop:​
Configure SentinelARC's Orchestrator to accept "Recovery Signals" from HealOps. For
example, if HealOps detects the MCP Fetch Server is down, it sends a signal to the
Orchestrator to switch that specific agent to a "Cached Data" mode.
